Core Linux Concepts

Linux is a powerful, open-source operating system. Here are the fundamental concepts every beginner should know:

File System
Linux uses a hierarchical file system, starting at the root directory (/).

Key directories: /home, /etc, /var, /usr.
Users & Permissions
Linux is multi-user. Each user has permissions defined by rwx (read, write, execute).

Example: chmod 755 file.txt
Shell
The command-line interface (CLI) where you interact with Linux using commands.

Popular shells: Bash, Zsh, Fish.
Processes
Programs running on your system. Managed via ps, top, kill.

Example: ps aux lists all processes.
Networking
Linux powers most servers. Key commands: ping, ifconfig, ssh.

Example: ssh user@host

Fun Fact: Linux runs 90% of the public cloud, all supercomputers, and most servers worldwide!

Essential Linux Commands

Master these basic commands to navigate and manage Linux like a pro:

Command Description Example
ls List directory contents. ls -l (detailed list)
cd Change directory. cd /home/user
pwd Print working directory (shows current location). pwd
mkdir Create a new directory. mkdir new_folder
rm Remove files/directories. rm file.txt or rm -r folder
cp Copy files/directories. cp file.txt /backup/
mv Move or rename files/directories. mv old.txt new.txt
cat Display file content. cat file.txt
grep Search for text in files. grep "error" log.txt
sudo Execute a command as superuser. sudo apt update

Pro Tip: Use man [command] (e.g., man ls) to read the manual page for any command!

Understanding File Permissions

Linux uses a robust permission system to control access to files and directories. Permissions are represented by rwx (read, write, execute) for user, group, and others.

# View permissions
ls -l

# Example output:
-rw-r--r-- 1 user group 1024 Jan 1 10:00 file.txt
|[-][rwx][r-x][r-x]
| | | | +-- Others: read-only
| | | +------- Group: read-only
| | +------------ User: read/write
| +---------------- File type (-=file, d=directory)

# Change permissions (chmod)
chmod 755 script.sh # User: rwx, Group/Others: r-x
chmod u+x script.sh # Add execute for user

# Change ownership (chown)
chown user:group file.txt
Permission Numeric Value Description
r 4 Read
w 2 Write
x 1 Execute
- 0 No permission

Example: chmod 644 file.txt → User: read/write, Group/Others: read-only.

Essential Linux Tools

These tools will help you master Linux and troubleshoot like a pro:

grep
Search for text patterns in files. Essential for log analysis and text processing.
Example: grep "error" /var/log/syslog
awk
Powerful text processing tool for extracting and formatting data.
Example: awk '{print $1}' file.txt
htop
Interactive process viewer. Better than top for monitoring system resources.
Install: sudo apt install htop
tmux
Terminal multiplexer. Lets you run multiple terminal sessions in a single window.
Basic commands: tmux new -s session_name, Ctrl+b d (detach)
tar
Archive files. Commonly used for compressing/decompressing files.
Example: tar -czvf archive.tar.gz /folder
cURL
Transfer data to/from a server. Useful for APIs and web requests.
Example: curl -O https://example.com/file.zip

Linux Cheat Sheet

Quick reference for common Linux tasks:

ls -l | grep ".txt"
List all .txt files in current directory.
find /home -name "*.conf"
Find all .conf files in /home.
du -sh * | sort -h
List directories by size (smallest to largest).
ps aux | grep "nginx"
Find all processes related to nginx.
df -h
Show disk space usage in human-readable format.
journalctl -xe
View system logs (systemd).
ssh user@remote_host
Connect to a remote server via SSH.
scp file.txt user@remote:/path
Securely copy a file to a remote server.

Next Steps in Your Linux Journey

You now have a solid foundation in Linux basics. Here’s how to keep improving:

  • Practice: Set up a Linux VM (VirtualBox) or use a Raspberry Pi.
  • Automate: Learn Bash scripting to automate repetitive tasks.
  • Explore: Try different distributions (Ubuntu, Fedora, Arch).
  • Contribute: Join open-source projects on GitHub.
  • Certify: Consider certifications like LPIC-1 or CompTIA Linux+.

Linux is a journey, not a destination. Keep experimenting, breaking things, and learning!
